# D1 Reference

## When To Use D1

Use D1 for lightweight relational data:

- report history
- user profiles
- usage counts
- payment/order records
- credit balance
- email/contact records

Do not use D1 for large generated assets, images, or logs with high write volume. Use R2 or external storage for those.

## Minimal Report Schema

```sql
CREATE TABLE IF NOT EXISTS reports (
  id TEXT PRIMARY KEY,
  created_at TEXT NOT NULL,
  method_id TEXT NOT NULL,
  method_name TEXT NOT NULL,
  question TEXT NOT NULL,
  concern_type TEXT,
  report_json TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_reports_created_at
ON reports (created_at DESC);
```

## Apply Schema

```powershell
npx wrangler d1 execute xuanxue --file=./schema.sql --remote
```

If the database is local during development:

```powershell
npx wrangler d1 execute xuanxue --file=./schema.sql --local
```

## Health Endpoint Pattern

```js
if (request.method === "GET" && url.pathname === "/api/health") {
  return sendJson({
    ok: true,
    service: "worker-api",
    storage: env.DB ? "d1" : "not-configured",
  });
}
```

## Insert Pattern

```js
await env.DB.prepare(
  `INSERT INTO reports (
    id, created_at, method_id, method_name, question, concern_type, report_json
  ) VALUES (?, ?, ?, ?, ?, ?, ?)`
```
